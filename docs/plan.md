# 智慧农业物联网边缘计算平台 - 5人小组两个月开发规划

## 🗓️ 总体时间安排
* **总周期**: 2个月（8周）
* **迭代周期**: 每2周一个Sprint，共4个迭代

## 👥 人员分工建议

| 角色 | 人数 | 主要职责 |
| :--- | :--- | :--- |
| 项目经理/组长 | 1人 | 项目协调、进度跟踪、文档管理、Git管理 |
| 后端开发 | 2人 | 后端API、数据库设计、业务 logique、设备通信 |
| 前端开发 | 1人 | Web界面开发、数据可视化、用户体验 |
| 测试部署 | 1人 | 测试用例、自动化测试、部署运维 |

## 📊 详细迭代规划

### 🚀 Sprint 1 (第1-2周)：需求分析与系统设计
* **目标**: 完成需求分析、技术选型、架构设计
* **主要任务**:
    * **需求分析 (全体)**
        * 智慧农业场景调研（温湿度监控、自动灌溉、光照控制等）
        * 用户故事地图绘制
        * 用例图、活动图绘制
    * **技术选型 (全体)**
        * 确定技术栈：建议Vue.js + Spring Boot + MySQL
        * 确定UML工具：推荐PlantUML
        * 确定协作平台：GitHub + Projects看板
    * **架构设计 (后端+组长)**
        * 四层架构设计：设备层、通信层、平台层、应用层
        * 数据库概念设计
        * 部署图设计
* **交付物**:
    * 需求分析文档
    * UML问题模型（用例图、活动图）
    * 技术选型报告
    * GitHub仓库初始化

### 🔧 Sprint 2 (第3-4周)：核心模块开发
* **目标**: 实现设备管理、数据采集基础功能
* **主要任务**:
    * **后端开发 (后端2人)**
        * 设备管理API（注册、状态、配置）
        * 数据接收接口
        * 数据库表结构实现
    * **前端开发 (前端1人)**
        * 设备管理界面
        * 基础数据展示
        * 响应式布局
    * **测试框架 (测试1人)**
        * 单元测试环境搭建
        * API测试用例
* **交付物**:
    * 设备管理功能可运行版本
    * 数据接收和存储功能
    * 基础Web界面

### 🎯 Sprint 3 (第5-6周)：业务功能实现
* **目标**: 实现自动化规则、数据可视化
* **主要任务**:
    * **自动化规则 (后端2人)**
        * 规则引擎设计
        * 条件判断逻辑
        * 控制指令下发
    * **数据可视化 (前端1人+测试1人)**
        * 实时数据图表
        * 历史数据查询
        * 告警信息展示
    * **集成测试 (测试1人)**
        * 端到端测试
        * 性能测试
* **交付物**:
    * 自动化规则配置界面
    * 数据可视化面板
    * 完整的业务流测试

### 🚢 Sprint 4 (第7-8周)：测试优化与部署
* **目标**: 系统测试、性能优化、部署上线
* **主要任务**:
    * **系统测试 (测试1人+全体)**
        * 集成测试用例执行
        * 用户验收测试
        * 安全测试
    * **性能优化 (后端2人+前端1人)**
        * 数据库优化
        * 前端性能优化
        * 缓存策略
    * **部署上线 (全体)**
        * Docker容器化
        * 云平台部署
        * 部署文档编写
* **交付物**:
    * 可部署的系统版本
    * 测试报告
    * 部署方案
    * 完整的实验报告

## 🛠️ 技术栈推荐
* **前端技术栈**
    * **框架**: Vue 3 + TypeScript
    * **UI库**: Element Plus
    * **图表**: ECharts
    * **构建工具**: Vite
* **后端技术栈**
    * **框架**: Spring Boot 3.x
    * **数据库**: MySQL 8.0 + Redis
    * **消息队列**: RabbitMQ（可选）
    * **API文档**: Swagger/OpenAPI
* **开发运维**
    * **版本控制**: GitHub + Git Flow
    * **项目管理**: GitHub Projects看板
    * **容器化**: Docker + Docker Compose
    * **部署**: 阿里云/腾讯云试用套餐
